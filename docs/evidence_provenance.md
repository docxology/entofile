# Evidence provenance — entofile

This release uses **zero mock substitution**, not "all inputs are real-world
datasets." The benchmark and security evidence is generated by real ENTO code
paths over documented inputs: committed fixture bytes, an explicit synthetic
throughput stress track, deterministic conformance vectors, and rendered
artifacts produced by the repository pipeline.

The claim is not that every input is a real-world observational dataset; the
claim is that every reported method output is produced by the repository's real
execution path and is signposted to its input class.
